# Contributing to UniMemory

Thank you for your interest in contributing to UniMemory! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: \`git clone https://github.com/shreyasgurav/UniMemory.git\`
3. Create a branch: \`git checkout -b feature/your-feature-name\`

## Development Setup

See the main [README.md](./README.md) for detailed setup instructions.

## Code Style

### Python (API)

- Follow PEP 8 style guide
- Use type hints where possible
- Format with Black (if configured)
- Maximum line length: 100 characters

### TypeScript/JavaScript (SDK & Dashboard)

- Use TypeScript for new code
- Follow ESLint rules
- Use Prettier for formatting
- Use meaningful variable and function names

## Making Changes

1. **Write clear commit messages**
   - Use present tense: "Add feature" not "Added feature"
   - First line should be a short summary (50 chars max)
   - Add more details in body if needed

2. **Test your changes**
   - Test locally before submitting
   - Ensure existing functionality still works
   - Add tests for new features if applicable

3. **Update documentation**
   - Update README.md if adding features
   - Update API documentation if changing endpoints
   - Add code comments for complex logic

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Update documentation as needed
3. Test your changes thoroughly
4. Submit PR with a clear description:
   - What changes were made
   - Why they were made
   - How to test them
   - Any breaking changes

5. Wait for review and address feedback

## Reporting Issues

When reporting issues, please include:

- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node/Python versions)
- Relevant logs or error messages
